# BodyParts3D 缓存机制说明

## 概述

本次优化为 BodyParts3D 网页应用添加了两层缓存机制，大幅提升后续访问的加载速度：

1. **Service Worker 缓存** - 缓存 HTTP 响应（OBJ 文件、JS、CSS 等）
2. **IndexedDB 缓存** - 缓存解析后的 3D 模型几何数据

## 缓存架构

```
┌─────────────────────────────────────────────────────────────┐
│                        用户请求                              │
└───────────────────────────┬─────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  1. 内存缓存 (Map) - 当前会话期间保持，最快的访问速度           │
│     └─ 已加载的 Three.js 对象                               │
└───────────────────────────┬─────────────────────────────────┘
                            │ 未命中
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  2. IndexedDB - 持久化存储，浏览器关闭后保留                   │
│     └─ 解析后的顶点、法线、UV、索引数据                       │
│     └─ 避免重复解析 OBJ 文件的开销                           │
└───────────────────────────┬─────────────────────────────────┘
                            │ 未命中
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  3. Service Worker Cache - HTTP 缓存层                      │
│     └─ 原始 OBJ 文件响应                                    │
│     └─ 静态资源 (JS, CSS, HTML)                             │
└───────────────────────────┬─────────────────────────────────┘
                            │ 未命中
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  4. 网络请求 - 最终回退                                     │
└─────────────────────────────────────────────────────────────┘
```

## 缓存策略

### 静态资源 (JS, CSS, HTML)
- **策略**: Cache First, then Network
- **行为**: 优先使用缓存，后台检查更新

### 3D 模型文件 (.obj)
- **策略**: Cache First with background update
- **行为**: 
  - 优先从缓存加载
  - 后台对比 ETag，如有更新则刷新缓存
  - 支持离线访问

### 解析后的几何数据
- **策略**: IndexedDB 持久化存储
- **行为**:
  - 首次加载后缓存解析结果
  - 后续访问直接重建 Three.js 对象，无需重新解析
  - 自动清理 30 天前的过期缓存

## 使用方法

### 正常访问
无需任何操作，浏览器会自动使用缓存加速加载。

### 查看缓存状态
点击右下角的 **「缓存就绪」** 指示器，可以查看：
- 磁盘缓存模型数量
- 内存缓存模型数量
- 缓存总大小
- 缓存命中率

### 清除缓存
点击缓存状态面板，确认后即可清除所有缓存。下次加载将重新下载模型。

### 强制更新缓存
如需强制获取最新版本的模型：
1. 点击缓存状态面板清除缓存
2. 刷新页面

## 浏览器兼容性

- **Service Worker**: Chrome 45+, Firefox 44+, Safari 11.1+, Edge 17+
- **IndexedDB**: 所有现代浏览器
- **降级处理**: 在不支持的浏览器中自动禁用缓存，不影响正常使用

## 性能提升

| 指标 | 首次加载 | 后续加载 (有缓存) |
|------|---------|------------------|
| 模型加载时间 | ~15-30秒 | ~3-8秒 |
| 网络请求数 | 202+ | 0 (离线可用) |
| 内存占用 | 基准 | +10-20% (内存缓存) |
| CPU 占用 (解析) | 高 | 极低 (跳过解析) |

*实际性能因设备和网络状况而异*

## 技术细节

### Service Worker (`sw.js`)
- 拦截所有 HTTP 请求
- 管理多个缓存存储：
  - `bodyparts3d-static-v1`: 静态资源
  - `bodyparts3d-models-v1`: 3D 模型文件
- 支持后台更新和缓存清理

### ModelCache 类 (`js/ModelCache.js`)
- 封装 IndexedDB 操作
- 内存缓存管理
- 统计信息收集
- 过期缓存清理

### 缓存键
- IndexedDB: 使用模型文件名作为键 (`FJ2772.obj` 等)
- 版本控制: 通过 `version` 字段管理缓存失效

## 注意事项

1. **隐私模式**: 某些浏览器的隐私模式可能限制 IndexedDB 使用
2. **存储限制**: 缓存占用约 50-100MB 磁盘空间
3. **更新检测**: 模型文件更新后，可能需要手动清除缓存或等待过期
